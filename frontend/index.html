Futures Cone

A governance decision-support simulator. Give it a proposal; it maps several plausible futures if it were accepted — never a score, never a recommendation, never a vote.

This tool never approves, rejects, ranks, or scores proposals. It generates multiple unranked plausible futures to inform human governance participants, run through nondeterministic LLM execution with GenLayer validator consensus.

plausible futures radiate outward — none is the predicted outcome

Write methods

Simulate a proposal

Paste a governance proposal, or try one of the examples below. Add a source link if this proposal already lives on Snapshot, Tally, or an on-chain governor.


  


  

    
Re-run as a variant

    
Take an already-simulated proposal and swap its percentage for a new value — e.g. re-run "increase by 8%" as "increase by 15%" without retyping the whole thing. Only works on proposals that contain a percentage.

    

      

        
        
      

      

        
        
      

      
    

    
  


  

    
Proposal length cap

    
Owner-only. Raises or lowers how long a proposal_text can be before the contract rejects it. Calling this from any account other than the one that deployed the contract will fail — that's the contract enforcing ownership, not an error in the page.

    

      

        
        
      

      
    

    
  


  

    
On-chain context grounding

    
Owner-only. Proof of concept, treasury category only. When enabled with a data source configured, simulate_proposal fetches real current treasury data (independently, by every validator) and grounds the LLM's reasoning in it instead of pure assumption. Disabled by default — both calls fail with a UserError unless you use the account that deployed the contract.
